## 오늘 마무리한 핵심 작업

### 1. 관리자 임시 비밀번호 발급

- 관리자 페이지 `/admin`의 사용자 관리 탭에서 회원별 `임시 비번` 발급 가능
- 서버 API: `/api/admin/reset-password`
- 관리자가 발급 버튼을 누르면 해당 회원 비밀번호가 임시 비밀번호로 초기화됨
- 임시 비밀번호는 발급 완료 화면에 한 번 표시되며, 운영자가 직접 사용자에게 전달

관련 파일:

- `app/admin/page.tsx`
- `app/api/admin/reset-password/route.ts`
- `lib/db.ts`

### 2. 회원 비밀번호 변경

- 로그인한 회원이 직접 비밀번호 변경 가능
- 임시 비밀번호로 로그인한 뒤 우측 상단 사용자 메뉴에서 `비밀번호 변경` 진입
- 현재 비밀번호 확인 후 새 비밀번호로 변경
- 변경 성공 시 자동 로그아웃 후 새 비밀번호로 다시 로그인해야 함

관련 파일:

- `app/auth/change-password/page.tsx`
- `app/api/user/change-password/route.ts`
- `app/page.tsx`

### 3. 비밀번호 찾기 페이지 운영 방식 변경

- 기존 `재설정 링크 받기` 이메일 발송 폼 제거
- 도메인 미구매 상태라 Resend의 `onboarding@resend.dev`는 일반 회원 이메일 발송에 제한이 있음
- 현재는 관리자에게 가입 이메일을 알려 임시 비밀번호를 받는 안내 페이지로 변경

관련 파일:

- `app/auth/forgot-password/page.tsx`

## 운영 절차

### 회원이 비밀번호를 잃어버린 경우

1. 회원이 가입 이메일을 운영자에게 전달
2. 운영자가 관리자 계정으로 로그인
3. `/admin` 접속
4. `사용자 관리` 탭에서 해당 회원 검색
5. `임시 비번` 버튼 클릭
6. 표시된 임시 비밀번호를 복사해서 회원에게 전달
7. 회원은 임시 비밀번호로 로그인
8. 우측 상단 사용자 메뉴에서 `비밀번호 변경`
9. 새 비밀번호로 변경

### 회원 횟수 충전

1. 관리자 계정 로그인
2. `/admin` 접속
3. `사용자 관리` 탭
4. 해당 회원의 `횟수 충전` 클릭
5. `+10`, `+50`, `+100` 같은 양수 입력 시 충전
6. 음수 입력 시 차감

값은 이 문서에 적지 않는다. 실제 값은 Vercel Project Settings의 Environment Variables에서 관리한다.

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `NEXTAUTH_SECRET`
- `NEXTAUTH_URL`
- `ADMIN_EMAIL`
- `ADMIN_PASSWORD`
- `INITIAL_CREDITS`
- `NEXT_PUBLIC_INITIAL_CREDITS`
- `NEXT_PUBLIC_SITE_URL`

- `ANTHROPIC_MODEL`

## 남은 개선 포인트

- Resend를 실제 도메인과 연결하면 이메일 비밀번호 재설정 기능을 다시 활성화 가능
- 결제 기능 추가 시 Stripe, 토스페이먼츠, PortOne 중 하나로 충전형 횟수 구매 연결 가능
- 국내 주식/지수/금/원자재용 표현을 더 자연스럽게 바꾸려면 `롱/숏` 대신 `상승 관점/하락 관점` 옵션 추가 권장
- 차트 자동 종목/타임프레임 인식 정확도는 실제 다양한 이미지로 계속 테스트 필요
- `app/page.tsx`, 일부 과거 파일의 깨진 한글 문자열은 기능에는 큰 문제 없지만 추후 전체 문구 정리 권장

## 주의사항

- `SUPABASE_SERVICE_ROLE_KEY`, `ANTHROPIC_API_KEY`, `RESEND_API_KEY`는 클라이언트 코드나 로그에 절대 출력하지 않는다.
- 횟수 차감은 반드시 서버 API에서만 처리한다.
- Vercel 서버리스에서는 로컬 파일 DB를 쓰면 안 된다. Supabase를 계속 사용한다.
- `tsconfig.tsbuildinfo`는 빌드 캐시이므로 수정되어도 보통 커밋하지 않는다.
